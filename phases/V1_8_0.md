# V1.8.0 — Branding customization

**Status:** shipped (2026-07-05)
**Branch:** `feat/v1.8.0-branding`
**Decision record:** DECISIONS 2026-07-05 (Phase 0 amendment to the v0.13
settings.toml read rule, operator-approved in the plan)

## Why

The product "face" was hardcoded: `bss-cli` + `$`/`●` in both portal
layouts, ~40 child-template titles, the phosphor palette duplicated
between `portal_base.css` and `_EMAIL_*` constants in
`bss-portal-auth/email.py`, the version number in header title lines,
and a separate `BSS_OPERATOR_NAME` env for the chat greeting. An
operator deploying BSS-CLI as a real MVNO could not put their own name
on it. v1.8 makes the face configuration: **operator name, theme
(color scheme), logo (glyph templates + optional image upload)** —
edited visually from the cockpit, stored in `.bss-cli/settings.toml`,
hot-reloaded everywhere.

## The shape

| Piece | Where | What |
|---|---|---|

## Themes

`phosphor` (default — byte-identical to the `portal_base.css` fallback,
test-pinned), `amber-crt`, `ice`, `magenta`, `paper` (mono),
`solarized-dark`. Dark-only: every component stylesheet assumes a dark
background; light themes need a component audit (deferred). A unit test
asserts WCAG contrast (fg/bg_elev ≥ 4.5, accent/bg ≥ 3.0, on_accent/
accent ≥ 3.0) for every palette.

## Doctrine added (CLAUDE.md "Branding (v1.8)")

- Read via `bss_branding.current()` at render/send time; never cached
  on `app.state`, never hardcoded in templates/emails.
- Palettes only in `THEMES`; no hex in `email.py` (guard); the CSS
  fallback stays in lockstep (test).
- Logo: magic-sniffed, size-capped by bytes read, fixed filenames,
  never SVG, browser portals only.
- Brand name/mark are operator input → `html.escape` at hand-built
  HTML seams; `validate_mark` rejects HTML-active chars.
- `bss-cli vX.Y.Z` footnote is product attribution — never rebranded.
- Three new `make doctrine-check` guards (email hex, template brand
  literals, settings.toml access outside the two sanctioned modules +
  the documented REPL `$EDITOR` carve-out).

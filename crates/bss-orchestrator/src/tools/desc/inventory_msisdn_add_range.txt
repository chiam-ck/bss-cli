v0.17 — bulk-extend the MSISDN pool by ``count`` contiguous numbers.

    The tool materializes the range as ``{prefix}{i:04d}`` for
    ``i in range(count)``. ``prefix`` MUST be 4-7 digits; the suffix
    is always 4 zero-padded digits, generated by the tool.

    Translating an operator's intent — IMPORTANT:

    - "add 50 numbers from 88880000" / "starting at 88880000"
        → prefix="8888", count=50 (NOT prefix="88880000"). The result
          is 88880000..88880049. Passing the full starting MSISDN as
          prefix is rejected by the policy (8 digits > the 7-digit
          maximum) and would otherwise produce 12-digit MSISDNs.
    - "add 1000 numbers under 9100"
        → prefix="9100", count=1000 (range 91000000..91000999).
    - "add the 88889999 number"
        → reject this shape: the tool only adds *contiguous ranges*.
          A single specific MSISDN is added by adding the prefix range
          that contains it (e.g. prefix="8888", count=10000).

    Operator-only: registered in the ``operator_cockpit`` profile, NOT
    in ``customer_self_serve``. Idempotent on overlap — overlapping
    ranges re-run cleanly; existing rows (including ``ported_out``)
    are preserved.

    Args:
        prefix: 4-7 digit numeric prefix, e.g. ``"9100"`` or ``"8888"``.
        count:  Numbers to add (1..10000).

    Returns:
        ``{prefix, count, inserted, skipped, first, last}``.

    Raises:
        PolicyViolation: ``msisdn.add_range.sane_prefix`` —
            non-digit prefix, prefix length outside [4, 7], or count
            outside [1, 10000].
